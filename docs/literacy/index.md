# 信息素养

信息素养是华工手册的底层能力：知道去哪里找信息，如何判断来源，如何把经验沉淀为后来者可用的公共知识。

## 不要被单一平台限制

很多校园信息被分散在公众号、群聊、朋友圈、学院通知、学生组织推文、旧网页和个人经验里。移动互联网平台方便传播，但也容易形成信息孤岛。本站鼓励把长期有价值的信息沉淀到开放网页、GitHub 仓库和可检索文档中。

不要长期只接受固定的信息来源，例如单一公众号、贴吧、知乎、某个 KOL 或某个群聊。社区基调、推荐算法、平台规则和个人立场都会影响你看到的信息。很多东西可以通过互联网、前人经验、实践和公共文档获得。

## 关键词

- GitHub：用 Issue 反馈问题，用 Pull Request 提交修改，用版本历史追踪变化。
- 开源：让项目的代码、内容结构和维护流程可被查看、复用和改进。
- 开放获取：尽可能指向公开、稳定、可访问的资料来源。
- Web：网页是更开放、可链接、可搜索的公共空间。
- 社区：寻找志同道合的人，不只依赖临时群聊。
- 邮件：正式、可追踪、跨平台的沟通方式。
- AI：可用于辅助整理、检查和搜索，但必须保留来源与人工判断。

这些关键词背后是一套更长期的能力：你可以自己寻找信息、判断信息、组织信息，也可以参与创造公共信息。

## 搜索技巧

建议掌握一些基本搜索技巧：

- 用关键词，而不是完整口语句子。
- 使用 `site:scut.edu.cn 推免` 搜索学校官网范围内的信息。
- 使用 `site:www2.scut.edu.cn 学院名 培养方案` 搜索学院官网信息。
- 使用 `filetype:pdf 学生手册` 搜索 PDF。
- 搜索政策时加年份，搜索经验时看发布时间。
- 对同一件事尽量找一手来源和多个独立来源交叉验证。

## GitHub 与 Markdown

如果你从未用过 GitHub，可以先理解三个动作：

- Issue：像公开工单，用来反馈问题、补充线索、讨论改动。
- Pull Request：把你修改过的内容提交给项目维护者审阅。
- Commit：每一次有说明的修改记录，方便追溯是谁在什么时候改了什么。

Markdown 是一种轻量文本格式。你不需要掌握复杂排版，只要会标题、列表、链接、表格，就可以贡献一篇可维护的页面。

```md
## 标题

- 一条信息
- 另一条信息

[来源标题](https://example.com)
```

## 邮件和正式沟通

微信和群聊适合即时沟通，但正式联系、投稿、申请、约见老师、咨询项目时，邮件更稳定、可追踪，也更容易跨平台保存。建议尽早注册并使用学生邮箱，学习基本邮件礼仪：

- 标题清楚说明主题。
- 开头说明身份和来意。
- 正文分段，问题具体。
- 附件命名清晰。
- 结尾留下姓名和联系方式。

## AI 的使用边界

AI 可以帮助你整理资料、生成提纲、检查错别字、提取表格、辅助搜索关键词，但不能替你判断事实是否真实。涉及政策、电话、费用、开放时间、申请截止日期、考试要求时，必须回到官方来源或原始材料核验。

## 判断信息

阅读任何信息时都先问：

- 它是一手信息，还是转载/总结/经验？
- 它发布于什么时候？
- 它适用于哪个校区、学院、专业、年级或身份？
- 它是事实、观点、建议，还是宣传？
- 是否能找到官方来源或多个独立来源交叉验证？

## 为后来者留下路标

华工手册的目标不是把所有信息变成“标准答案”，而是把前人走过的路、找过的入口、踩过的坑沉淀为后来者能继续修正的路标。

这也是华工手册的价值底色：世界上的很多东西是由具体的人创造出来的，我们也可以参与建设它。

如果你发现某条信息过期，不需要等“更专业的人”来修。你可以提交 Issue、发邮件、开 PR，或者把新来源发给维护者。公共知识库就是在这些小修小补中变得可靠。
